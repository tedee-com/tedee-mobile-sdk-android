openLock

abstract fun openLock(param: Byte = BluetoothConstants.PARAM_NONE)

Sends an OPEN_LOCK (0x51) command to unlock the lock. The lock's response is provided through the ILockConnectionListener.onIndication method. The response is a ByteArray representing the lock's result, structured as follows: [COMMAND, COMMAND_RESULT_STATUS].

For example, a successful response is 51, 00 when is success, where:

  • 51 indicates the OPEN_LOCK command.

  • 00 means the command completed successfully.

Parameters

param

The OPEN_LOCK command can use additional parameters:

  • NONE (0x00): Unlocks the lock. This is the default setting.

  • AUTO (0x01): Unlock from Auto Unlock feature.

  • FORCE (0x02): Forces lock to unlock the lock till jam.

  • WITHOUT_PULL (0x03): Opens the lock without Pull Spring, if configured. For more information, visit Tedee Lock BLE API documentation.
